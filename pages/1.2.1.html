1.2.1 Best Practice Methodology - Quick Notes

Problems have 3 parts: cause, symptom, consequence. Fix the consequence first (impact on user), then the cause.

CompTIA's 6-Step Troubleshooting Model

  1. Identify the problem - gather info from the user, note any recent changes, back up if needed, and start documentation.
  2. Establish a theory of probable cause - question the obvious and research if needed.
  3. Test the theory - if confirmed, plan next steps; if not confirmed, form a new theory or escalate.
  4. Plan and implement the solution - follow vendor instructions.
  5. Verify full functionality - add preventive measures if applicable.
  6. Document everything - findings, actions, outcomes, and lessons learned.
