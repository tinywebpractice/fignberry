3.2.5 Redundant Array of Independent Disks (RAID)

What is RAID?

RAID (Redundant Array of Independent/Inexpensive Disks) distributes data across multiple disks to provide fault tolerance and improve performance. It sacrifices some disk capacity to achieve redundancy.

Note: To the OS, a RAID array appears as a single storage volume that can be partitioned and formatted like any other drive. RAID can also stand for "Redundant Array of Independent Devices."

Why Use RAID?

  • A failed boot drive causes the system to crash
  • A failed data drive can result in permanent data loss if no backup exists
  • RAID mitigates these risks through redundancy and/or performance improvements

RAID Implementation Types

Type How It Works Notes
Software RAID Implemented using OS features No dedicated hardware required; uses CPU resources
Hardware RAID Uses a dedicated RAID controller adapter card Drives connect to SATA ports on the controller, not the motherboard; often supports hot-swapping
Integrated RAID Built into the motherboard chipset A middle-ground option; no separate adapter card needed

Common RAID Levels

RAID Level Min. Drives Method Fault Tolerance Use Case
RAID 0 2 Striping None — any disk failure = total data loss Maximum performance; no redundancy
RAID 1 2 Mirroring One disk can fail; data survives on mirror Critical data; OS drives
RAID 5 3 Striping with parity One disk can fail Balance of performance, capacity, and redundancy
RAID 10 4 RAID 1 + RAID 0 (nested) One disk per mirrored pair can fail High performance + redundancy

Note: RAID levels are numbered 0–6. Entry-level controllers may only support RAID 0 or 1; mid-level controllers may add RAID 5 and RAID 10.

Key Terms

  • RAID – Redundant Array of Independent/Inexpensive Disks; distributes data across multiple drives for fault tolerance and/or performance
  • Fault tolerance – The ability of a system to continue operating even if one component fails
  • Striping (RAID 0) – Data is split across multiple drives for speed; no redundancy
  • Mirroring (RAID 1) – Data is duplicated across two drives for redundancy
  • Parity (RAID 5) – Error-checking data distributed across drives; allows recovery if one drive fails
  • Nested RAID (RAID 10) – Combines two RAID levels (RAID 1 + RAID 0) for both performance and redundancy
  • Hot-swapping – Replacing a failed drive without shutting down the system; supported by hardware RAID
  • RAID controller – A dedicated adapter card that manages RAID drives; drives connect to its SATA ports
