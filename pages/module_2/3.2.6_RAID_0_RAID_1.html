3.2.6 RAID 0 and RAID 1

Choosing a RAID Level

When selecting a RAID level, consider these four factors:

  • Fault tolerance – How many disks can fail without data loss?
  • Read/write performance – Does the array need to be fast?
  • Capacity – How much usable storage is needed?
  • Cost – How many disks are required?

Important: Always use disks that are identical in capacity, type, and performance. If disks differ in size, the smallest disk determines the maximum usable space across the array.

RAID 0 – Striping without Parity

Data is divided into blocks and distributed across all disks in the array, allowing multiple disks to service requests in parallel for improved performance.

Property Detail
Minimum disks 2
Usable capacity Sum of all drive capacities (limited by smallest disk)
Fault tolerance None — if any disk fails, the entire volume fails
Performance Best read/write performance of all RAID levels
Best used for Non-critical cache storage where speed is the priority

Warning: RAID 0 provides no redundancy. A single disk failure causes a system crash and requires data recovery from backups.

RAID 1 – Mirroring

Every write operation is duplicated on a second disk simultaneously. Read operations can use either disk, slightly boosting read performance.

Property Detail
Minimum disks 2
Usable capacity 50% of total disk space (one disk is the mirror)
Fault tolerance One disk can fail; the other takes over with minimal impact
Write performance Slight overhead — data must be written to both disks
Read performance Slightly improved — either disk can serve reads
Rebuild speed Faster than parity-based RAID levels
Best used for Critical data; OS drives where redundancy is the priority

Note: After a disk fails it should be replaced quickly to restore redundancy. The new disk is automatically populated with data from the remaining disk, though performance is temporarily reduced during the rebuild.

RAID 0 vs RAID 1 Comparison

Feature RAID 0 RAID 1
Method Striping Mirroring
Min. drives 2 2
Fault tolerance None 1 disk failure
Usable capacity 100% of all drives 50% of total drives
Performance Highest Slight write overhead; good read speed
Cost efficiency Best Most expensive per GB

Key Terms

  • Striping (RAID 0) – Data is split into blocks and spread across multiple disks for performance; no fault tolerance
  • Mirroring (RAID 1) – Data is written identically to two disks simultaneously; protects against one disk failure
  • Fault tolerance – The ability to survive a disk failure without losing data
  • Rebuild – The process of restoring data to a replacement disk after a failure
  • Disk striping – Dividing data into blocks and distributing them across all disks for parallel access
