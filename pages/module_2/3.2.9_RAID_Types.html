3.2.9 RAID Types

Hardware RAID vs Software RAID

Type How It's Set Up When It Runs Options
Hardware RAID Configured via a dedicated RAID controller Before the OS boots More options and control over RAID types
Software RAID Configured through the operating system (e.g. Disk Management in Windows) Within the OS Fewer options; no dedicated controller needed

Configuring Software RAID in Windows

Software RAID in Windows is configured through Disk Management. Drives must be unallocated before being added to an array.

Creating a Striped Volume (RAID 0)

  • Right-click an unallocated drive and select New Striped Volume
  • Add the desired drives to the stripe and click Next
  • Assign a label (e.g. "Striped") and complete the wizard
  • Data is split across all drives — each drive does less work, improving performance
  • The usable volume size is the combined capacity of all drives in the stripe

Creating a Mirrored Volume (RAID 1)

  • Right-click an unallocated drive and select New Mirrored Volume
  • Add one additional drive to the mirror and click Next
  • Assign a label (e.g. "Mirrored") and complete the wizard
  • All data written to the drive is copied to the second drive automatically
  • The usable volume size is the size of one drive only — the second drive is hidden and used purely for redundancy

Dynamic Disks

When setting up software RAID in Windows, drives are converted to dynamic disks.

  • Dynamic disks are required for software RAID configurations in Windows
  • You cannot install an operating system on a dynamic disk — only use dynamic disks for non-boot drives
  • If Windows is already installed, converting remaining drives to dynamic disks is safe

What You See in File Explorer

Volume Type Visible Size in File Explorer Reason
Striped (RAID 0) Combined size of all drives All drives contribute to the total usable space
Mirrored (RAID 1) Size of one drive only The second drive is hidden — it is a copy used for redundancy only

Key Terms

  • Hardware RAID – RAID configured by a dedicated controller before the OS boots; more options and control
  • Software RAID – RAID configured within the OS (e.g. Windows Disk Management); fewer options
  • Striped Volume (RAID 0) – Data split across multiple drives for performance; no redundancy
  • Mirrored Volume (RAID 1) – Data duplicated across two drives for redundancy; usable space = one drive
  • Dynamic disk – A Windows disk type required for software RAID; cannot be used to install an OS
  • Disk Management – The Windows tool used to configure partitions, volumes, and software RAID arrays
