3.3.1 System RAM and Virtual Memory

How RAM Fits in the Memory Hierarchy

When a process runs or a file opens, the data is loaded from the fixed disk into system RAM. The CPU then fetches instructions from RAM into its cache and registers as needed, managed by a memory controller.

Storage Type Speed Volatile?
CPU Registers / Cache Fastest Yes
RAM (System Memory) Very fast Yes — data lost without power
SSD (Flash memory) Fast No
HDD (Magnetic) Slowest No

Note: The amount of RAM determines a PC's ability to handle multiple applications simultaneously and process large files efficiently. RAM is measured in gigabytes (GB).

Address Space – Data vs Address Pathways

The bus connects the CPU, memory controller, and RAM. It has two key pathways:

Pathway Function Typical Width
Data pathway Determines how much data is transferred per clock cycle 64 bits (single channel)
Address pathway Determines how many memory locations the CPU can track; limits max physical and virtual memory 32-bit or 48-bit (on 64-bit CPUs)

32-bit vs 64-bit CPU Memory Limits

CPU Type Address Bus Max Memory Virtual Memory Address Space
32-bit CPU 32-bit Up to 4 GB 4 GB (with paging to mass storage)
64-bit CPU 48-bit (typical) Up to 256 TB 256 TB (with paging to mass storage)
64-bit CPU (theoretical) 64-bit Up to 16 exabytes Not used in practice

Note: Both 32-bit and 64-bit CPUs use a 64-bit data bus to transfer data between the CPU and RAM. It is the address bus width that limits how much memory can be accessed.

Virtual Memory and Paging

When physical RAM runs low, the OS uses virtual memory — a portion of the hard drive or SSD is used as an extension of RAM. This process is called paging. Data is swapped between RAM and mass storage as needed. Paging is much slower than using physical RAM.

Key Terms

  • RAM (Random Access Memory) – Volatile system memory; faster than SSDs and HDDs; stores active data and instructions
  • Memory controller – Manages the flow of data between the CPU and RAM
  • Bus – The communication system connecting the CPU, memory controller, and RAM
  • Data pathway – The part of the bus that transfers data; typically 64 bits wide
  • Address pathway – The part of the bus that identifies memory locations; determines max addressable memory
  • Virtual memory – An extension of RAM using mass storage (HDD/SSD) when physical RAM is full
  • Paging – The process of moving data between RAM and mass storage to support virtual memory
  • 32-bit CPU – Limited to 4 GB of addressable memory due to 32-bit address bus
  • 64-bit CPU – Uses a 48-bit address bus in practice; supports up to 256 TB of memory
